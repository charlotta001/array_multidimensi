ai answer   : Yes! There are two main ways to create a thread in Java.
            Way 1 — Extend the Thread class
            You create a class that extends Thread, then override the run() method. Whatever you put inside run() is what that worker will do.
            Then you create an object of that class and call .start() to begin the worker.
            Way 2 — Use Runnable (more common)
            You create a Runnable — think of it as a task description — and pass it to a Thread object. Then call .start().
            This is more flexible and more commonly used in real projects.
            The key methods you need to know:
            .start() → tells the thread to begin working
            .run() → the actual task the thread does (you define this)
            .setDaemon(true) → makes the thread automatically die when your main program ends, so it doesn't keep running in the background forever
            Thread.sleep(ms) → pauses the current thread for X millisecond
